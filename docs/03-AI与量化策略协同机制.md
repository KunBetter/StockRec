# AI 与量化策略协同机制

> 鲸灵宝选股系统 | 2026-05-23

---

## 一、策略架构总览

鲸灵宝的选股引擎由两大子系统组成：**量化策略引擎** 和 **DeepSeek AI 分析层**。二者各司其职，协同工作。

```
输入：全A股 K线数据 + 财务数据 + 资金流向 + 新闻舆情
                    │
    ┌───────────────┴───────────────┐
    │                               │
    ▼                               ▼
量化策略引擎 (90%)          DeepSeek AI (10%)
├─ Layer1 多因子 (60%)      ├─ 财报解读
├─ Layer2 ML预测 (30%)      ├─ 新闻情绪分析
├─ Layer3 事件驱动 (10%)     ├─ 行业趋势判断
│                               └─ 综合推荐理由生成
    │                               │
    ▼                               ▼
 复合评分排序 ──────┬────── 情绪分 + AI 摘要
                    │
                    ▼
            最终推荐清单 (Top 30)
```

---

## 二、量化策略引擎 —— 核心主力

量化策略是选股的**核心引擎**，负责输出数学上可验证的排序结果。

### Layer 1：多因子模型（权重 60%）

基于 20+ 个量化因子，对全市场股票进行横截面评分：

| 因子类别 | 因子示例 | 含义 |
|---------|---------|------|
| 价值因子 | EP、BP、SP、股息率 | 估值便宜程度 |
| 质量因子 | ROE、ROA、毛利率、资产周转率 | 企业盈利能力 |
| 成长因子 | 营收增速、利润增速、研发占比 | 业绩增长潜力 |
| 动量因子 | 1月动量、3月动量、12-1月动量 | 价格趋势强度 |
| 反转因子 | 周反转、日内反转 | 短期超买超卖 |
| 波动因子 | 历史波动率、下行波动率 | 风险水平 |
| 流动性因子 | 换手率、Amihud非流动性 | 交易活跃度 |
| 资金因子 | 主力净流入、北向资金、融资变化 | 资金面强弱 |

**处理流程**：
1. 各因子独立计算 → 截面 z-score 标准化 → 去极值（Winsorize 3σ）
2. ICIR 加权（过去 12 个月信息系数稳定性越高，权重越大）
3. 行业中性化 + 市值中性化（剔除风格暴露）
4. 输出 Layer1 综合得分（0-100）

### Layer 2：机器学习预测（权重 30%）

基于 LightGBM 的监督学习模型：

- **特征**：252 日量价时序特征（收益率、波动率、成交量变化等）+ 财务特征 + 资金流特征
- **目标**：未来 20 日收益率
- **训练**：时间序列交叉验证，日频滚动重训练
- **输出**：预期收益率排序

### Layer 3：事件驱动（权重 10%）

检测高确定性的事件信号：

| 事件类型 | 权重 | 触发条件 |
|---------|:---:|---------|
| 业绩超预期 | 60% | 净利润同比增速 > 20% |
| 高股息预案 | 30% | 股息率 > 4% |
| 股票回购 | 10% | 公司发布回购公告 |

### 三层融合

```
composite_score = Layer1 × 0.6 + Layer2 × 0.3 + Layer3 × 0.1
                         ↓
                  风险分级 (低/中/高)
                         ↓
                   Top 30 推荐清单
```

**特点**：纯数据驱动、可回测验证、客观一致、高频率自动化

## 三、DeepSeek AI 分析层 —— 语义增强

AI 分析在量化结果之上**叠加语义理解和定性判断**，弥补纯粹数字模型的盲区。

### 四大分析维度

| 分析模块 | 输入 | 输出 | 作用 |
|---------|------|------|------|
| **财报解读** | 最新季报财务数据 | 盈利能力评价、成长性判断、财务健康度 + overall_score | 发现数字背后的经营质量 |
| **新闻情绪** | 近期个股新闻标题/摘要 | 情绪标签（正面/负面/中性）+ sentiment_score | 捕捉市场即时情绪 |
| **行业趋势** | 行业分类 + 行业动态 | 景气度判断 + 趋势展望 + 竞争地位 | 自上而下校验个股逻辑 |
| **综合推荐** | 上述所有 + 量化得分 | 投资建议摘要 + 推荐等级 + 风险提示清单 | 生成人类可读的最终结论 |

### 调用方式

- **模型**：DeepSeek-chat (Pro)
- **接口**：OpenAI 兼容 SDK → `https://api.deepseek.com/v1`
- **频率**：每小时对 Top 30 股票增量分析
- **并发**：最多 3 个并发请求（semaphore 控制）

### AI 输出如何融入最终评分

```
AI 情绪分 = f(推荐等级, 财报评分, 新闻情绪)
         ├─ "强烈推荐" → 85
         ├─ "推荐"     → 70
         ├─ "中性"     → 50
         └─ "谨慎"     → 30

最终 composite_score = predicted_return × 0.5
                     + momentum_score  × 0.2
                     + quality_score   × 0.2
                     + AI情绪分        × 0.1  ← 10% 权重
```

## 四、量化 + AI 协同效应

### 各司其职

| 维度 | 量化策略 | DeepSeek AI |
|------|---------|------------|
| **核心作用** | 定方向：哪些股票值得看 | 验逻辑：为什么值得看 |
| **权重占比** | 90%（主导排序）| 10%（微调得分）|
| **输出形式** | 数字排名与得分 | 文字理由 + 情绪分 + 风险提示 |
| **核心优势** | 可回测、客观一致、高频自动化 | 可解释、灵活应变、理解语义 |
| **核心局限** | 依赖历史规律、对突发事件盲区 | 不稳定、有幻觉风险、延迟较高 |

### 量化做不到的，AI 补充

| 量化盲区 | AI 补充方式 |
|---------|------------|
| 财报附注中的隐藏风险（如存货减值、担保风险）| 读取财报文本，识别异常信号 |
| 突发新闻的即时影响（如政策变动、高管离职）| 新闻情感分析，判断利好/利空程度 |
| 行业政策变化的深度解读 | 分析政策文本，评估对景气度的中长期影响 |
| 推荐理由的用户沟通 | 综合所有量化+定性数据，输出简明摘要 |

### AI 做不到的，量化保障

| AI 局限 | 量化兜底 |
|---------|---------|
| 输出不稳定（同样问题多次回答可能不同）| 数学模型结果可复现 |
| 幻觉风险（可能编造不存在的信息）| 仅输入客观数据，不做外部知识推断 |
| 无法批量处理全市场（每次调用需数秒）| 可分钟级处理 800+ 只股票 |
| 缺乏统计显著性验证 | 因子 IC/ICIR 长期跟踪 |

### 协同案例

> **万科 A** — 量化模型预测收益 +26.8%，综合得分 70
>
> DeepSeek 分析后标注：
> - 财报亮点：无（财务数据不足）
> - 行业判断：地产短期承压，政策边际宽松可能带来修复机会
> - 风险提示：高波动风险、政策不确定性、需求端压力
> - 综合建议：**中性**
>
> → 情绪分仅 50 分（而非盲目追高）

> **比亚迪** — 量化综合得分 85.5，预测收益 +3.1%
>
> DeepSeek 分析后标注：
> - 行业判断：新能源汽车高景气度，政策利好持续
> - 竞争地位：行业龙头，规模优势明显
> - 风险提示：价格竞争加剧、补贴退坡
> - 综合建议：**推荐**
>
> → 情绪分 70 分，确认量化判断

## 五、设计原则

1. **量化定锚，AI 验锚** — AI 不推翻量化排序，而是在量化基础上做语义校验和微调
2. **低权重介入** — AI 情绪分仅占 10%，避免 AI 不稳定性主导决策
3. **增量更新** — 量化引擎每日重跑全量，AI 仅对变更显著的股票做增量分析
4. **可追溯** — 每次推荐同时存储量化得分明细和 AI 分析全文，支持回溯审查
5. **降级容错** — AI 不可用时，系统以默认情绪分（50）继续运行，不影响核心选股
