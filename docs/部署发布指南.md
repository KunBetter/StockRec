# StockRec 部署与发布指南

## 前置条件

- 阿里云实名认证账号
- 已注册域名（需 ICP 备案，周期约 20 天）
- iOS 开发者账号（$99/年）+ Android 应用签名证书
- DeepSeek API Key

---

## 一、阿里云资源开通

所有资源放同一 VPC 和可用区（推荐华东 1/华东 2），安全组规则：
- 公网：80/TCP、443/TCP（面向用户）
- 内网：5432（RDS）、6379（Redis）仅允许 ECS 安全组访问

### 1.1 ECS 云服务器

| 配置 | 推荐值 |
|------|--------|
| 实例规格 | ecs.c7.xlarge（4C8G）或 ecs.g7.xlarge |
| 系统盘 | 40GB ESSD PL1 |
| 操作系统 | Ubuntu 22.04 LTS 或 CentOS 7.9+ |
| 计费方式 | 包年包月（¥406/月，新用户首年 5-6 折） |

操作：控制台 → 云服务器 ECS → 创建实例 → 记录公网 IP

### 1.2 RDS PostgreSQL

| 配置 | 推荐值 |
|------|--------|
| 版本 | PostgreSQL 15 |
| 规格 | 2C4G |
| 存储 | 100GB ESSD PL1 |
| 计费 | 包年包月（¥900/月） |

操作：创建后设置白名单（ECS 内网 IP），获取内网连接地址 `pgm-xxx.rds.aliyuncs.com`

### 1.3 Redis

| 配置 | 推荐值 |
|------|--------|
| 版本 | Redis 7.0 |
| 规格 | 1GB 标准版 |
| 计费 | 包年包月（¥256/月） |

操作：创建后获取内网连接地址 `r-xxx.redis.rds.aliyuncs.com`

### 1.4 对象存储 OSS

| 配置 | 推荐值 |
|------|--------|
| Bucket 名称 | `stockrec-data` |
| 存储类型 | 标准存储 |
| 计费 | 按量（约 ¥15/月） |

### 1.5 容器镜像服务 ACR

创建个人版实例，命名空间 `stockrec`，用于存放 Docker 镜像。

## 二、ECS 环境初始化

SSH 登录 ECS 后执行：

```bash
# 安装 Docker
curl -fsSL https://get.docker.com | bash
sudo usermod -aG docker $USER

# 安装 Docker Compose
sudo apt-get update && sudo apt-get install -y docker-compose-plugin

# 创建目录结构
sudo mkdir -p /opt/stockrec /opt/backups
sudo chown -R $USER:$USER /opt/stockrec /opt/backups
```

## 三、首次部署

### 3.1 配置环境变量

在 `/opt/stockrec/.env` 中写入以下内容（替换占位符为实际值）：

```bash
DATABASE_URL=postgresql+asyncpg://stockrec:<RDS密码>@<RDS内网地址>:5432/stockrec
DATABASE_URL_SYNC=postgresql://stockrec:<RDS密码>@<RDS内网地址>:5432/stockrec
REDIS_URL=redis://<Redis内网地址>:6379/0
JWT_SECRET_KEY=<生成命令: openssl rand -hex 32>
DEEPSEEK_API_KEY=<DeepSeek API Key>
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
OSS_ENDPOINT=oss-cn-hangzhou.aliyuncs.com
OSS_BUCKET=stockrec-data
OSS_ACCESS_KEY_ID=<阿里云 AccessKey ID>
OSS_ACCESS_KEY_SECRET=<阿里云 AccessKey Secret>
```

### 3.2 部署代码

```bash
cd /opt/stockrec

# 方式一：从 GitHub 拉取
git clone git@github.com:<your-org>/StockRec.git .
cp docker/docker-compose.yml .  # compose 文件放根目录

# 方式二：直接 rsync 本地代码
# rsync -avz --exclude 'node_modules' --exclude '.venv' \
#   ./ root@<ECS公网IP>:/opt/stockrec/
```

### 3.3 启动服务

```bash
cd /opt/stockrec

# 构建并启动所有容器（PostgreSQL + Redis + App + Nginx）
docker compose -f docker/docker-compose.yml up -d --build

# 执行数据库迁移
docker compose exec app alembic upgrade head

# 迁移存量数据（如果有 SQLite 数据）
docker compose exec app python scripts/migrate_sqlite_to_pg.py

# 健康检查
curl http://localhost:8000/api/v1/health
# 期望返回: {"status":"ok","version":"0.1.0",...}
```

### 3.4 安装 Nginx + HTTPS

```bash
# Ubuntu
sudo apt-get install -y nginx certbot python3-certbot-nginx

# 配置 Nginx（参考 docker/nginx.conf）
sudo cp docker/nginx.conf /etc/nginx/sites-available/stockrec
sudo ln -s /etc/nginx/sites-available/stockrec /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx

# DNS A 记录指向 ECS 公网 IP 后，申请 SSL 证书
sudo certbot --nginx -d api.stockrec.example.com
```

证书自动续期：
```bash
# certbot 自动配置了定时任务，验证：
sudo certbot renew --dry-run
```

## 四、CI/CD 自动部署

### 4.1 GitHub Secrets 配置

仓库 Settings → Secrets and variables → Actions → New repository secret：

| Secret 名 | 内容 | 获取方式 |
|-----------|------|---------|
| `ECS_HOST` | ECS 公网 IP | 阿里云控制台 |
| `ECS_USER` | SSH 用户名 | `root` 或自建用户 |
| `ECS_SSH_KEY` | SSH 私钥 | `cat ~/.ssh/id_ed25519` |

### 4.2 部署流程

```
Git Push (main) → GitHub Actions
  → docker build → push ACR
  → SSH into ECS
  → docker compose pull → up -d
  → alembic upgrade head
  → health check（失败自动回滚）
```

### 4.3 手动触发

Actions 页面 → Deploy → Run workflow

## 五、React Native App 构建与发布

### 5.1 开发环境

```bash
cd mobile

# 安装依赖
npm install

# iOS 依赖
cd ios && pod install && cd ..

# 开发模式运行
npx react-native run-ios      # iOS 模拟器
npx react-native run-android   # Android 模拟器
```

### 5.2 生产 API 地址配置

编辑 `mobile/src/services/config.ts`，替换生产 API 地址：

```typescript
const config: AppConfig = {
  apiBaseUrl: "https://api.stockrec.example.com/api/v1",
  wsBaseUrl: "wss://api.stockrec.example.com/ws",
};
```

### 5.3 iOS 发布

1. Xcode 打开 `mobile/ios/StockRec.xcworkspace`
2. 选择 `Product → Archive`
3. Organizer → Distribute App → App Store Connect
4. App Store Connect 中完成元数据、截图、隐私政策
5. 提交审核

### 5.4 Android 发布

```bash
cd mobile/android

# 生成签名密钥（仅首次）
keytool -genkey -v -keystore stockrec.keystore \
  -alias stockrec -keyalg RSA \
  -keysize 2048 -validity 10000

# 配置签名（android/app/build.gradle 中配置 signingConfigs）

# 构建 AAB
./gradlew bundleRelease

# 上传到：
# - Google Play Console（海外）
# - 应用宝 / 华为应用市场 / 小米应用商店（国内）
```

### 5.5 Android 签名配置

在 `android/gradle.properties` 中添加：

```
STOCKREC_UPLOAD_STORE_FILE=stockrec.keystore
STOCKREC_UPLOAD_KEY_ALIAS=stockrec
STOCKREC_UPLOAD_STORE_PASSWORD=<密码>
STOCKREC_UPLOAD_KEY_PASSWORD=<密码>
```

## 六、数据库运维

### 备份

RDS PostgreSQL 默认开启自动备份（7 天保留期）。额外手动备份：

```bash
# ECS 上执行
docker compose exec postgres pg_dump -U stockrec stockrec \
  > /opt/backups/stockrec_$(date +%Y%m%d_%H%M%S).sql

# 同时上传到 OSS
docker compose exec app python -c "
import boto3, os
from datetime import datetime

# 使用 oss2 或其他 OSS SDK
"
```

### 恢复

```bash
docker compose exec -T postgres psql -U stockrec stockrec \
  < /opt/backups/stockrec_20260101_120000.sql
```

### 查看日志

```bash
# App 日志
docker compose logs -f app --tail=100

# Nginx 访问日志
docker compose logs -f nginx --tail=50

# 数据库慢查询日志
docker compose exec postgres tail -f /var/lib/postgresql/data/log/postgresql-*.log
```

## 七、监控告警

### 阿里云控制台

- RDS 控制台 → 监控与报警 → 设置 CPU > 80% / 存储 > 85% 告警
- ECS 控制台 → 云监控 → CPU/内存/磁盘告警

### Sentry 错误追踪

```bash
# 后端添加 Sentry SDK
pip install sentry-sdk

# main.py 中添加
import sentry_sdk
sentry_sdk.init(dsn="<Your Sentry DSN>")
```

### 应用指标

在 `/opt/stockrec/` 下运行 Prometheus + Grafana（可选，需要额外 Docker 容器）：

```yaml
# docker-compose 中添加
prometheus:
  image: prom/prometheus
  volumes:
    - ./prometheus.yml:/etc/prometheus/prometheus.yml

grafana:
  image: grafana/grafana
  ports:
    - "3000:3000"
```
